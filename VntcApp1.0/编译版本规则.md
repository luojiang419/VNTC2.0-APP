# 编译版本规则

## 当前起始版本

- 当前编译版本：以 `scripts/build_version.txt` 当前内容为准
- 当前版本源文件：`scripts/build_version.txt`

## 自动递增规则

1. `scripts/build_version.txt` 中保存“本次要编译出的版本号”，例如 `2.1`。
2. 执行 `scripts/export_portable_package.ps1` 时：
   - 产物文件名会使用当前版本号，例如 `VNT_App_2.1_Windows_Portable.zip`
   - 软件产品名固定为 `VNTC APP2.0`
   - 软件窗口标题会同步为当前版本号，例如 `VNTC APP2.0 v2.1`
   - 编译成功后，`scripts/build_version.txt` 自动递增 `0.1`，变为下一次编译版本
3. 执行 `scripts/export_installer_package.ps1` 时：
   - 会复用同一次版本号生成便携包和安装包
   - 安装包文件名会使用当前版本号，例如 `VNT_App_2.1_Windows_Setup.exe`
   - 安装器和快捷方式显示名称固定为 `VNTC APP2.0`
   - 卸载项显示名称会同步为当前版本号，例如 `VNTC APP2.0 v2.1`
   - 全部成功后，`scripts/build_version.txt` 自动递增 `0.1`
4. 如果编译失败，不写回下一版本，避免版本号空跳。

## 标题与文件名来源

- 软件产品名：由 `APP_PRODUCT_NAME` 构建参数注入，固定为 `VNTC APP2.0`
- 软件窗口标题：由 `APP_WINDOW_TITLE` 构建参数注入，格式为 `VNTC APP2.0 vX.Y`
- 便携包文件名：`VNT_App_X.Y_Windows_Portable.zip`
- 安装包文件名：`VNT_App_X.Y_Windows_Setup.exe`
- 安装器 / 开始菜单 / 桌面快捷方式显示名：`VNTC APP2.0`
- 卸载项显示名：`VNTC APP2.0 vX.Y`

## 手动调整方法

- 如果要指定下一次编译版本，直接修改 `scripts/build_version.txt`
- 例如把文件内容改为 `2.5`，下一次成功编译就会产出 `v2.5`，随后自动推进到 `2.6`
