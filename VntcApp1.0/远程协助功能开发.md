# 远程协助功能开发

## 目标

为当前 Windows 版 VNT App 增加“远程协助”功能，基于独立维护的 `vntcrustdesk` 集成版 RustDesk，通过 VNT 虚拟 IP 直连发起远程控制，不依赖官方公网中继。

## 当前仓库已完成的实现

### 1. 应用内功能

- 新增左侧栏/底部导航“远程协助”菜单。
- 新增页面：`lib/pages/remote_assist_page.dart`
- 新增远程协助核心模块：`lib/remote_assist/`
- 新增运行时能力：
  - `RemoteAssistManager`
  - `RemoteAssistLauncher`
  - `RemoteAssistHealthService`
  - `RemoteAssistPresenceService`
  - `RemoteAssistLog`
- Presence 采用 UDP `49998`，用于补齐“名称 - 虚拟 IP”展示。
- 控制连接目标固定为 `<peerVirtualIp>:49999`。
- 已明确切掉旧的 runtime/便携/内嵌目录方案，不再沿用旧思路。
- 新增运行时检测增强：
  - 不再只依赖 `remote_assist/vntcrustdesk_manifest.json`
  - 若 manifest 缺失，会继续通过 Windows 服务和卸载注册表定位 `vntcrustdesk`
- 新增应用内自修复能力：
  - 当检测到当前目录携带 `vntcrustdesk.msi + bootstrap_vntcrustdesk.ps1` 时，点击“安装/修复”会触发管理员提权并自动补装/修复 `vntcrustdesk`
  - 若仅检测到独立安装但未绑定当前应用，会在页面明确提示“已检测到独立安装（未绑定当前应用）”

### 2. 安装器接入

- 新增 `scripts/bootstrap_vntcrustdesk.ps1`
  - 在主安装流程中静默安装 `vntcrustdesk.msi`
  - 启动服务
  - 创建基础防火墙规则
  - 写入 `remote_assist/vntcrustdesk_manifest.json`
- 新增 `scripts/uninstall_vntcrustdesk.ps1`
  - 只卸载由当前安装器托管的 `vntcrustdesk`
  - 移除远程协助防火墙规则
- 新增 `scripts/stage_vntcrustdesk_artifact.ps1`
  - 将独立 fork 仓库产出的 MSI 复制到当前仓库约定目录
- 修改 `scripts/export_installer_package.ps1`
  - 安装包现在会强制检查 `third_party/vntcrustdesk/windows/dist/vntcrustdesk.msi`
  - 在 Inno 安装器中嵌入 MSI 和 bootstrap/uninstall 脚本
- 修改 `scripts/export_portable_package.ps1`
  - 便携包现在也会携带：
    - `remote_assist/artifacts/vntcrustdesk.msi`
    - `scripts/bootstrap_vntcrustdesk.ps1`
    - `scripts/uninstall_vntcrustdesk.ps1`
  - 便携包首次进入“远程协助”页面后，可通过“安装/修复”按钮完成提权安装

### 3. 产物约定

- 独立 fork 仓库不并入当前仓库。
- 当前仓库只消费以下产物：
  - `third_party/vntcrustdesk/windows/dist/vntcrustdesk.msi`
  - `third_party/vntcrustdesk/windows/dist/vntcrustdesk.version.json`

## 新链路说明

1. 独立 `vntcrustdesk` fork 仓库输出 MSI。
2. 使用 `scripts/stage_vntcrustdesk_artifact.ps1` 复制 MSI 到当前仓库产物目录。
3. 执行 `scripts/export_installer_package.ps1`。
4. Inno 安装器先安装 VNT App，再静默安装 `vntcrustdesk.msi`。
5. bootstrap 脚本写入 manifest，供应用运行时定位安装路径、版本和服务信息。
6. 远程协助页面通过健康检查、Presence 和服务状态来发起连接。

## 仍依赖外部完成的部分

以下内容必须在独立 `vntcrustdesk` fork 仓库完成，当前仓库无法单独替代：

- 可执行文件名改为 `vntcrustdesk.exe`
- 服务名改为 `vntcrustdesk`
- 默认直连端口改为 `49999`
- 默认开启 direct-server / approve-mode 等定制行为
- 与官方 RustDesk 的配置目录、安装目录、服务和卸载项彻底隔离
- 关闭自动更新与官方公共服务器依赖
