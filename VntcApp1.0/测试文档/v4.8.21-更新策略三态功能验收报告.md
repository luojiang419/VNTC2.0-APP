# v4.8.21 更新策略三态功能验收报告

## 功能结果

设置页面新增三种持久化更新策略：

1. 手动更新：启动或点击立即检查后自动检测并下载安装包，下载完成后弹窗提示安装。
2. 自动更新：自动检测和下载；Windows 直接启动独立静默更新器，Android 自动拉起系统安装器。
3. 关闭更新：启动和手动检查入口均在网络请求前结束，不检测、不下载。

品牌配置 `updateEnabled=false` 仍是最高优先级总开关：入口隐藏、更新会话拒绝、服务副作用前阻断。

## 关键行为验证

- 未保存策略时默认手动更新。
- 三种策略可持久化，未知值安全回退到手动更新。
- 单次启动只检测和下载一次。
- 没有新版时不下载。
- 手动模式下载完成后按钮为“立即安装”，不会重复下载。
- 自动模式无需应用内二次确认即进入安装链路。
- 自动安装失败时保留已下载安装包并回退到安装提示。
- 关闭模式的检测调用数和下载调用数均为 0。
- 静默启动时，手动模式的提示会等窗口可见后再显示。

## 平台说明

- Windows 支持全自动静默安装：拉起独立更新器、退出旧进程、静默覆盖并重启新版。
- 普通 Android 应用受系统安全模型限制，应用可自动下载并拉起安装器，但系统仍可能要求用户确认安装来源或最终安装授权。
