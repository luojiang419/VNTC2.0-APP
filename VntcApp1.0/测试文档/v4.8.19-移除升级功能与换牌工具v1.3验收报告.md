# v4.8.19 移除升级功能与换牌工具 v1.3 验收报告

## 验收结论

功能已完成并通过端到端验收。换牌工具 v1.3 新增“移除升级功能（推荐，默认勾选）”，默认生成的客户包不会自动检查更新、不显示软件更新入口、不弹升级窗口、不会访问发布页或执行下载/安装/更新会话。

## 工具行为

1. GUI 包含“导入源安装包”、“输入新名称”、“添加新图标（可选）”、“移除升级功能”、“隐藏关于页”和“重新打包”。
2. “移除升级功能”在编译后的 GUI 中确认为 `Checked=true`；“隐藏关于页”为 `false`。
3. CLI 不传 `--remove-update` 时默认移除；显式传 `--remove-update false` 或 `0` 才保留。
4. 工具仍为单一 EXE，内置 Inno Setup、rcedit、简体中文语言文件和相关许可证，客户无需安装额外打包环境。
5. 图标仍支持 ICO、PNG、JPG/JPEG 和 BMP；过大图片会自动缩放、压缩并生成 Windows 多尺寸 ICO。

## 运行时防护

- 设置页不构造“软件更新”项及对应分隔线。
- 关于页不构造升级卡片，版本号保留显示但不可点击。
- 启动自动检查、待显示弹窗、手动检查弹窗和“发现新版本”弹窗均在入口层阻断。
- 检查、下载、打开安装包、打开发布页、启动静默更新器、执行更新会话六条服务路径均在副作用前统一拒绝。
- 完整 `--run-update-session` 参数在应用入口处被阻断，记录日志后显式退出，不会留下无窗口进程。
- `branding.json` 损坏或缺失时优先从母版清单恢复策略；非官方进程在两份配置都不可用时安全关闭升级。官方 `vnt_app.exe` 未配置品牌时仍保留升级。

## 兼容与分发边界

- 完整移除升级要求 v4.8.19+且母版宣布 `removeUpdateFeature`；旧母版不允许伪造成功。
- 取消“移除升级”后会恢复官方 GitHub 升级，仅应在已建立同品牌更新流程时使用。
- 隐藏关于页不免除 LICENSE、NOTICE 和第三方许可声明义务。
- 换牌后文件哈希会变化，商业代码签名应在最终换牌后执行。当前产物未使用商业证书，部分电脑首次运行可能显示 SmartScreen 提示。
