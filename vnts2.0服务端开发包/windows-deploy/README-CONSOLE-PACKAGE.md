# VNTS 2.0 Windows x64 增强控制台

本发布包面向配置较高、需要实时可视化监控与完整业务管理的 Windows 主机。入口是 `VNTS2-Console.exe`。增强版是独立项目：默认服务名为 `vnts2-console`，默认管理端口为 `39871`，默认隧道端口为 `39872`，只使用当前增强版解压目录内的 `data`。它不会读取、迁移、覆盖或停止轻量版的 `vnts2` 服务及其 `data`，两个版本可以同时运行。

## 首次使用

1. 完整解压 ZIP，不要单独复制 EXE 或 DLL。
2. 双击 `VNTS2-Console.exe`，同意 Windows 管理员权限提示。
3. 控制台会自动创建便携配置、安装并启动同目录服务，无需再点击“安装”。全新配置会立即显示全屏首次设置门禁。
4. 设置管理员账号、密码和确认密码；保存后控制台自动备份配置、重启服务并使用新凭据验证，通过后才进入软件。
5. 在“设置 → 关闭与开机行为”选择默认关闭动作和开机自启方式，再到“安全与隐私”按需调整自动锁定时长和快捷键。
6. 检查默认网段、监听器和可选 WireGuard/互联配置。

## 关闭与开机行为

- 默认点击关闭按钮时最小化到系统托盘，VNTS2 服务继续运行；单击托盘图标可恢复窗口。
- 托盘菜单和设置页提供“关闭服务并退出”，会先安全停止 Windows 服务，再结束控制台。
- 默认关闭行为可改为“关闭服务并退出”，保存后标题栏关闭按钮立即按新行为执行。
- 开机行为支持不开机自启、登录后显示主窗口、登录后静默驻留托盘。
- 开机自启使用 `VNTS2-Console-Autostart` 最高权限登录计划任务；移动解压目录后再次启动控制台会自动更新任务路径。

## 安全约束

- 管理 API 只允许回环 HTTP(S) 地址；不要把 Web 管理绑定到公网。
- 全新配置使用不可预测的一次性引导凭据并写入首次设置标记；用户设置成功后该标记会删除，引导凭据立即失效。
- API 密码不限制长度，但不能为空、不能为 `admin`、不能与管理员账号相同。
- 控制台不持久化登录密码、会话 Cookie、CSRF Token、WireGuard 一次性私钥或服务端 Token。
- 未认证或锁定时不会构建侧栏和业务页面；默认无操作 2 小时锁定，默认快捷键为 `Ctrl+Shift+L`。
- WireGuard 生成式配置只显示一次，请立即安全保存；不要截图或上传二维码。
- 配置保存前会在 `data/.backups` 创建时间戳备份。
- 正式发布包只含 `config.example.toml`，不含真实 `data/config.toml`、数据库、证书、密钥或日志。

## 便携目录

增强控制台的 Flutter 运行资源和服务可变数据共同位于同级 `data`。该目录只属于增强版，不能与轻量版共用。迁移或备份时必须复制整个增强版解压目录；卸载 `vnts2-console` Windows 服务不会删除 `data`。

## 故障处理

- 首次准备失败：确认整个 ZIP 已完整解压、当前账号同意管理员权限、目录未被其他同名 `vnts2` 服务占用；随后在准备面板重试。
- 无法连接 API：先在“服务运维”查看服务状态并运行诊断，再确认配置中的 Web 管理已启用且绑定回环地址。
- 认证失败：使用当前管理员账号和密码重新登录；控制台不会从磁盘读取或恢复明文密码。
- 配置错误：使用 `data/.backups` 中最近的 `config.toml.pre-flutter-*.bak` 恢复后重启。
- 发布包完整性：用根目录 `SHA256SUMS.txt` 校验内部文件，并用 ZIP 旁的 `.sha256` 校验下载包。
