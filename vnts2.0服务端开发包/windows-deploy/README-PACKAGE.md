# VNTS 2.0 Windows x64 发布包

本目录是 VNTS 2.0 Windows 服务端的离线交付包。包内不含真实密码、token、密钥、证书、数据库、日志或历史备份。

轻量版是独立项目：默认服务名为 `vnts2`，默认管理端口为 `29871`，默认隧道端口为 `29872`，只使用当前轻量版解压目录内的 `data`。它不会读取、迁移、覆盖或停止增强版的 `vnts2-console` 服务及其 `data`，两个版本可以同时运行。

## 图形界面快捷使用

双击 `VNTS2-Manager.exe`，在 Windows 权限提示中选择“是”，即可打开原生 C# WinForms 服务管理器。它是正式客户入口，不会显示命令行窗口。GUI 提供：

- 关闭窗口默认最小化到系统托盘，服务继续运行；托盘菜单可恢复窗口或关闭服务并退出。
- 右上角“关闭行为”可立即选择动作，“偏好设置”可保存默认关闭动作。
- 开机行为支持不开机自启、登录后显示主窗口、登录后静默驻留托盘；自启通过最高权限计划任务运行。

- 五分区结构化配置设置（基础网络、Web 管理、WireGuard、服务器互联、高级安全）；
- 一键安装并启动；
- 启动、停止、刷新状态和运行诊断；
- 网络管理、QUIC 地址显示和快捷复制；
- 打开内嵌 Web 控制台；
- 确认后卸载服务；
- 自定义隔离服务名；
- 深色/浅色主题切换和运行日志查看。

GUI 使用 Windows 自带的 .NET Framework WinForms，不需要安装额外桌面运行库。PowerShell 脚本仅作为隐藏的后台服务运维组件，不是客户入口；卸载不会删除运行数据。

所有可变运行数据都位于同级 `data` 目录，包括 `config.toml`、日志、数据库、密钥、证书和 `.backups`。首次点击“配置”或“安装并启动”会从安全模板自动初始化 `data/config.toml`。配置按钮会打开专业设置弹窗，不需要使用记事本或手工理解 TOML；保存前会自动备份原配置。需要迁移时，卸载服务后复制整个文件夹，再在新路径中通过 GUI 安装即可。

如果同名服务已经安装在其他目录，GUI 会显示“已有部署”以及原程序/配置路径，并将主按钮切换为“迁移并启动服务”。只有用户在确认框选择“是”后，GUI 才会校验并备份旧文件，将配置和运行数据迁入当前包的 `data`，再更新服务启动命令。若要从当前解压目录安装并行测试实例，请先在“服务名”中填写未占用的独立名称并刷新状态。

若当前服务尚未配置 Web 管理端，点击“Web 控制台”并确认后，GUI 会仅绑定本机 `127.0.0.1:29871`、生成 24 位强密码、备份并原子更新配置，然后重启服务。登录信息会在独立窗口中显示并可手动复制，密码不会出现在操作日志中。

## 完整性校验

在当前目录执行：

```powershell
Get-FileHash -Algorithm SHA256 .\vnts2.exe
Get-Content .\SHA256SUMS.txt
```

`MANIFEST.json` 记录产品版本、平台、架构、文件长度和 SHA-256；`SHA256SUMS.txt` 覆盖除自身外的全部包内文件。

## 首次部署

1. 完整解压 ZIP，不要只取出单个 EXE。
2. 双击 `VNTS2-Manager.exe`，同意 Windows 管理员权限提示。
3. 点击“配置”，通过设置页确认监听地址、虚拟网段和其他功能。
4. 点击“仅保存”，或在服务已安装时选择“保存并启动/重启”。
5. 首次部署可直接点击“安装并启动”，程序会自动初始化安全配置并注册服务。

默认模板未启用 Web 管理端和 WireGuard，因此不包含真实密码、token 或密钥。启用相关设置时，请使用弹窗内的强密钥生成功能，并妥善保管客户端配置。

## WireGuard 一键生成客户端配置

先打开 GUI 的“配置 → WireGuard”，启用 WireGuard，填写主密钥文件、UDP 监听地址，并设置外部设备实际可以访问的域名或公网 IP 与端口。

服务启动后进入 Web 控制台的 WireGuard Peer 页面，点击“新增 Peer”，默认即可一键生成客户端密钥、自动分配虚拟 IP，并获得可复制、下载或扫码导入的完整配置。标准 WireGuard 公钥是 32 字节密钥的 Base64 表示，固定为 44 个字符；管理员不需要记忆或手工输入，一键生成流程会自动处理。私钥和完整客户端配置只在本次结果页面显示，请在关闭前保存。

## 命令行安装和管理服务

使用管理员 PowerShell：

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\install-vnts2-service.ps1
.\status-vnts2-service.ps1
.\diagnose-vnts2-service.ps1
```

常用操作：

```powershell
.\start-vnts2-service.ps1
.\stop-vnts2-service.ps1
.\uninstall-vnts2-service.ps1
```

安装脚本会把当前根目录的 `vnts2.exe` 和 `data/config.toml` 注册为 Windows 服务，并把 ACL 收紧为 SYSTEM 与 Administrators。请保留包内目录结构，不要只移动单个 EXE。

卸载服务不会删除 `data` 中的任何配置或运行数据；迁移或销毁数据必须由管理员单独确认。

## Web 管理端

推荐直接使用 GUI 的“Web 控制台”按钮安全启用并打开管理端，也可在“配置 → Web 管理”中设置账号和强密码。管理端只允许回环地址，并从服务器本机访问 `http://127.0.0.1:29871/`。
