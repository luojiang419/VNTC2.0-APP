# 聊天室双机联调说明

## 目标

用于 `VntcApp1.0` 在两台 Windows 客户端之间联调聊天室首版能力：

- 在线发现
- 好友申请
- 私聊文字
- 图片发送
- 文件发送
- 语音消息
- 一对一语音

## 运行前准备

1. 两台 Windows 机器都加入同一个 VNT 网络。
2. 两台机器都使用调试构建运行：
   - `scripts/run_windows.bat`
3. 两台机器都允许麦克风访问。
4. 检查 Windows 防火墙是否拦截当前进程。
5. 打开房间页 -> 聊天室。
6. 在两端聊天室页顶部的 `联调工具` 卡片中点击：
   - `清空聊天数据`
   - `刷新发现`

## 联调顺序

### 1. 在线发现

- A、B 都进入聊天室页。
- 检查两端 `在线设备` 列表：
  - 能看到对方
  - 不会看到自己

### 2. 好友流程

- A 点击 B 的 `加好友`
- B 在 `好友` 列表看到 `等待处理`
- B 点击 `通过`
- A、B 双方状态都变成好友

### 3. 私聊文字

- A 点击 B 的 `私聊`
- 发送一段文字
- B 应能收到文字
- A 消息状态应从 `发送中/已发送` 进入 `已送达`

### 4. 图片发送

- A 发送一张图片
- B 应看到 `待接收`
- B 点击 `接收`
- 图片预览应成功显示

### 5. 文件发送

- A 发送一个普通文件
- B 点击 `接收`
- 文件应能落地到聊天附件目录

### 6. 语音消息

- A 按住录语音
- 松开发送
- B 接收后点击 `播放语音`

### 7. 一对一语音

- A 点击右上角电话图标呼叫 B
- B 接听
- 双方确认能互相听到
- 任一方挂断
- 状态应正确回收

## 异常场景补测

### 附件拒绝

- B 拒绝 A 的图片或文件
- A 应看到状态变成 `已拒绝`

### 附件过期

- B 长时间不处理附件
- 消息状态应转为 `已过期`

### 拉黑

- B 拉黑 A
- A 再发私聊/附件/语音邀请
- B 侧不应进入新会话或新请求

### 断开连接

- 任一方断开 VNT
- 在线状态和语音状态应回收

## 联调诊断

聊天室页已内置 `联调工具`：

- `刷新发现`
  - 强制重跑在线发现
- `查看诊断`
  - 显示当前网络、虚拟 IP、数据库路径、附件路径、日志路径、语音包计数
  - 可直接复制
- `清空聊天数据`
  - 清空聊天数据库
  - 清空附件缓存
  - 清空频道本地状态
  - 清空聊天室调试日志

## 日志位置

- 聊天室调试日志文件：`chat-debug.log`
- 实际路径可在 `查看诊断` 中直接查看和复制

## 建议排查顺序

1. 先看 `查看诊断` 中两端是否拿到了正确虚拟 IP 和网络 key
2. 再对照两端 `chat-debug.log`
3. 若在线发现异常，先查 hello / profile_sync
4. 若附件异常，先查 offer / accept / header / sha256
5. 若语音异常，先查 callInvite / callAccept / 音频初始化 / PCM 包计数
