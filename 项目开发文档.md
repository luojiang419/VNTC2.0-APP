# 项目开发文档

## 1. 项目概览

本项目是一个 VNT 2.0 管理项目，当前主要包含两套 Linux Web 控制面板和一套 Windows Flutter 桌面端：

- `VNTS 2.0` 服务端控制台：管理 `vnts2.service`，维护 `/root/vnts2/config.toml`。
- `VNTC 2.0` 客户端控制台：管理 `vntc2.service`，维护 `/root/vntc2.0/config.toml`，并通过 `vnt2_ctrl` 读取客户端运行态。
- `VNTC 2.0 Windows` 桌面端：基于 Flutter + Rust，直接集成官方 VNT 核心，面向 Windows 客户端连接场景。

Linux 控制面板整体使用 Python 标准库实现后端 HTTP 服务，不依赖 `pip`、`npm` 或数据库；前端为原生 HTML/CSS/JavaScript；Windows 桌面端则采用 Flutter + Rust 架构，目标是原生 Windows 客户端连接与多实例管理。

## 3. 后端入口结构

### 3.1 VNTS 服务端面板入口

- 入口文件：`vnts2.0/server.py`
- 主模块：`vnts2.0/vnt_panel/web.py`
- 默认地址：`0.0.0.0:2223`
- 默认管理服务：`vnts2.service`
- 默认配置文件：`/root/vnts2/config.toml`

`vnts2.0/server.py` 只负责导入并执行 `vnt_panel.web.main()`。实际 HTTP 服务、路由分发、静态资源服务、认证校验、配置读写与 systemd 操作都在 `vnts2.0/vnt_panel/` 内完成。

### 3.2 VNTC 客户端面板入口

- 入口文件：`vntc2.0/client_server.py`
- 主模块：`vntc2.0/vntc_panel/web.py`
- 默认地址：`0.0.0.0:2224`
- 默认管理服务：`vntc2.service`
- 默认配置文件：`/root/vntc2.0/config.toml`
- 默认控制工具：`/root/vntc2.0/vnt2_ctrl`

`vntc2.0/client_server.py` 只负责导入并执行 `vntc_panel.web.main()`。客户端面板复用同项目内 `vnt_panel.auth.AuthManager` 和 `vnt_panel.system.ServiceManager`，并额外使用 `vntc_panel.runtime.VNTClientRuntimeManager` 读取客户端运行态。

### 3.3 VNTC Windows 桌面端入口

- Flutter 入口：`vntc2.0windows/lib/main.dart`
- Rust 桥接入口：`vntc2.0windows/rust/src/api/vnt_api.rs`
- Windows Runner：`vntc2.0windows/windows/runner/main.cpp`
- Windows 依赖资源：`vntc2.0windows/windows/dlls/`

该项目当前保留官方 `vnt-dev/VntApp` 的 Flutter + Rust 桌面底层接入方式，但已移除官方界面层，改为按 `vntc2.0/static_client/` 的 Linux 控制台结构重建 Windows 首页。Flutter 负责 UI、配置持久化和多连接管理，Rust 直接调用官方 `vnt` 核心库。Windows Runner 已声明管理员权限，并在运行时自动复制匹配架构的 `wintun.dll`。

## 4. 后端模块职责

### 4.1 `vnts2.0/vnt_panel/`：VNTS 服务端面板

```text
vnt_panel/
├── __init__.py
├── auth.py             # 登录认证、会话 Token、账号密码持久化
├── configuration.py    # VNTS config.toml 的读取、结构化解析、校验、渲染与备份
├── settings.py         # VNTS 面板环境变量与默认配置
├── system.py           # systemctl、journalctl、ps 封装
└── web.py              # HTTP 服务、API 路由、静态文件服务、日志 SSE
```

核心功能：

- 登录、退出、会话检测。
- 账号密码修改并写入认证文件。
- 读取服务状态、进程 PID、CPU、内存、运行时长。
- 启动、停止、重启 `vnts2.service`。
- 读取 `journalctl` 日志快照和 SSE 实时日志。
- 读取、保存结构化 TOML 配置。
- 直接保存原始 TOML 配置。
- 写入配置前自动备份已有配置文件。

### 4.2 `vntc2.0/vntc_panel/`：VNTC 客户端面板

```text
vntc_panel/
├── __init__.py
├── configuration.py    # VNTC config.toml 的读取、结构化解析、校验、渲染与备份
├── runtime.py          # 调用 vnt2_ctrl 读取 info、ips、clients、route
├── settings.py         # VNTC 面板环境变量与默认配置
└── web.py              # HTTP 服务、API 路由、静态文件服务、日志 SSE
```

核心功能：

- 登录、退出、会话检测。
- 账号密码修改并写入认证文件。
- 读取、启动、停止、重启 `vntc2.service`。
- 读取 `journalctl` 日志快照和 SSE 实时日志。
- 读取、保存结构化客户端 TOML 配置。
- 直接保存原始 TOML 配置。
- 写入配置前自动备份已有配置文件。
- 调用 `vnt2_ctrl -p <ctrl_port>` 获取 `info`、`ips`、`clients`、`route` 运行态。

### 4.3 `vntc2.0windows/`：VNTC Windows 桌面端

```text
vntc2.0windows/
├── lib/              # Flutter UI、配置模型、多连接状态管理
├── rust/             # flutter_rust_bridge 桥接与 VNT Rust 核心接入
├── rust_builder/     # Windows FFI 插件构建层
├── windows/          # Runner、CMake、UAC 提权、wintun.dll 资源
└── assets/           # 图标与日志配置
```

核心功能：

- 保存多套网络配置，并选择默认连接项。
- 同时创建多个 VNT 连接实例，每个实例可配置独立 `device_id`、网卡名、端口组和服务器地址。
- 启动时自动按系统架构选择并复制 `wintun.dll`。
- 构建出的 Windows Runner 自动申请管理员权限。
- 主界面视觉和版式改为贴近 Linux 客户端控制台，而不是使用官方 `VntApp` 风格。

- 高级 TOML：直接编辑原始 TOML。

### 5.2 VNTC 前端：`vntc2.0/static_client/`


主要页面区域：

- 登录弹层。
- 总览：服务状态、虚拟 IP、连接状态、对端数量、运行时长、控制端口、服务端列表。
- 连接配置：`network_code`、`server`、`ip`、`password`、`cert_mode`、`device_name`、`ctrl_port`、`mtu`、压缩/纠错/穿透相关开关、路由映射列表等。
- 运行态：展示 `vnt2_ctrl` 读取到的客户端信息、对端列表与路由信息。
- 高级 TOML：直接编辑原始 TOML。
- 实时日志：通过 `EventSource` 连接 `/api/logs/stream`。
- 设置：修改面板登录账号与密码。

## 6. API 功能结构

两套面板的 API 风格基本一致，差异是 VNTC 多了运行态接口。

### 6.1 公共接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/session` | 检查当前 Cookie 会话 |
| `POST` | `/api/login` | 登录并设置 HttpOnly Cookie |
| `POST` | `/api/logout` | 退出登录并清除 Cookie |
| `GET` | `/api/overview` | 获取面板元信息、服务状态、配置、账号摘要 |
| `GET` | `/api/settings/account` | 获取账号摘要 |
| `PUT` | `/api/settings/account` | 修改账号密码 |
| `GET` | `/api/status` | 获取 systemd 服务状态 |
| `POST` | `/api/service` | 执行 `start`、`stop`、`restart` |
| `GET` | `/api/logs?lines=200` | 获取日志快照 |
| `GET` | `/api/logs/stream?lines=80` | SSE 实时日志 |
| `GET` | `/api/config` | 获取配置快照 |
| `PUT` | `/api/config/structured` | 保存结构化配置，可选择保存后重启服务 |
| `PUT` | `/api/config/raw` | 保存原始 TOML，可选择保存后重启服务 |

### 6.2 VNTC 专属接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/runtime` | 调用 `vnt2_ctrl` 获取客户端运行态 |

## 7. 配置与数据流

### 7.1 VNTS 配置流

```text
vnts2.0/static/app.js
  -> /api/config 或 /api/config/structured
  -> vnts2.0/vnt_panel/web.py
  -> vnts2.0/vnt_panel/configuration.py
  -> /root/vnts2/config.toml
  -> /root/vnts2/.backups
```

配置管理器会使用 `tomllib` 解析 TOML，用结构化字段维护常用配置，并在写入前复制旧配置到备份目录。

### 7.2 VNTC 配置流

```text
vntc2.0/static_client/app.js
  -> /api/config 或 /api/config/structured
  -> vntc2.0/vntc_panel/web.py
  -> vntc2.0/vntc_panel/configuration.py
  -> /root/vntc2.0/config.toml
  -> /root/vntc2.0/.backups
```

客户端配置管理器维护连接服务器、虚拟网配置、证书模式、控制端口、MTU、穿透/路由相关开关与映射列表。

### 7.3 运行态与服务控制流

```text
Web UI
  -> /api/status 或 /api/service
  -> ServiceManager
  -> systemctl / journalctl / ps
```

VNTC 额外有运行态读取：

```text
Web UI
  -> /api/runtime
  -> VNTClientRuntimeManager
  -> vnt2_ctrl -p <ctrl_port> info|ips|clients|route
```

## 9. 部署结构

### 9.1 systemd 模板

systemd 模板已按项目归档：

- `vnts2.0/deploy/vnt-panel.service.example`：VNTS 控制面板服务，执行 `/usr/bin/python3 /opt/vnt-panel/server.py`。
- `vntc2.0/deploy/vntc-panel.service.example`：VNTC 控制面板服务，执行 `/usr/bin/python3 /opt/vnt-panel/client_server.py`。
- `vntc2.0/deploy/vntc2.service.example`：VNTC 客户端核心服务，执行 `/root/vntc2.0/vnt2_cli --conf /root/vntc2.0/config.toml`。

## 11. 安全与权限注意事项

- 面板会调用 `systemctl`、`journalctl` 和 `ps`，部署时通常需要 root 权限或等效授权。
- 登录 Cookie 使用 `HttpOnly` 和 `SameSite=Strict`。
- 密码不会明文保存，认证文件保存 PBKDF2-HMAC-SHA256 哈希与 salt。
- 静态文件服务会限制路径必须位于对应静态目录内，防止路径穿越读取任意文件。
- 配置保存前会解析 TOML，原配置存在时会自动备份。
- 如果面板暴露公网，应修改默认账号密码，并配合防火墙或 HTTPS 反向代理。

## 12. 后续开发建议

- 新增功能优先判断影响范围：VNTS 专属、VNTC 专属，或两套面板可复用。
- VNTC 项目内保留了运行所需的 `vnt_panel/auth.py` 与 `vnt_panel/system.py` 副本，修改共享认证或服务管理逻辑时需要同步两边。
- 配置字段改动需要同步后端配置管理器、前端表单、渲染逻辑和测试。
- 客户端运行态改动需要同步 `vntc2.0/vntc_panel/runtime.py` 的解析逻辑和 `vntc2.0/static_client/app.js` 的展示逻辑。
- 发布包内容变化时，需要检查 `项目交付包/release_bundle/build_release_bundle.py` 的复制清单。
- 每次功能修改后按项目规范补充测试，并生成新的进度快照。
